Weenix Operating System

A complete Unix-like operating system kernel built from scratch as part of Brown's Operating Systems course

C x86 Assembly Systems Programming Kernel Development Virtual Memory

Note: Due to academic integrity policies, source code for this project cannot be publicly shared.

Overview

Weenix is a semester-long project that involves building a fully functional Unix-like operating system kernel from the ground up. This intensive project is considered one of the most challenging and rewarding experiences in Brown's Computer Science curriculum.

The kernel implements all the fundamental components of a modern operating system, providing hands-on experience with low-level systems programming and deep understanding of how operating systems work under the hood.

Implementation Phases

The project was completed in four major phases, each building upon the previous:

1. Threads & Processes

Implemented the foundation of the kernel with thread management and process control.

  • Thread creation, scheduling, and synchronization
  • Process management with fork(), exec(), and wait()
  • Mutex locks and condition variables
  • Priority-based scheduler implementation

2. Device Drivers

Created drivers for hardware interaction and terminal I/O.

  • TTY (terminal) driver with line discipline
  • Keyboard and display drivers
  • Interrupt handling and device management
  • Character and block device interfaces

3. Virtual File System (VFS)

Built a complete file system layer with multiple file system support.

  • VFS abstraction layer
  • S5FS (System V File System) implementation
  • File operations: open, read, write, seek, close
  • Directory operations and path resolution

4. Virtual Memory

Implemented complete virtual memory management with demand paging.

  • Page fault handling and demand paging
  • Copy-on-write (COW) optimization
  • Memory-mapped files
  • Anonymous and file-backed memory regions

Key Challenges & Learning

Building an OS from scratch presented unique challenges that deepened my systems programming expertise:

  • Debugging at the kernel level: Developed skills in using GDB for kernel debugging without standard debugging tools
  • Race conditions: Mastered synchronization primitives and lock ordering to prevent deadlocks
  • Memory management: Gained deep understanding of virtual memory, page tables, and TLB management
  • System architecture: Learned to design clean interfaces between kernel subsystems

Skills Developed

This project provided invaluable experience in:

  • Low-level C programming and x86 assembly
  • Concurrent programming and synchronization
  • Memory management and virtual memory systems
  • File system design and implementation
  • Hardware/software interface and interrupt handling
  • Debugging complex systems-level code
  • Reading and understanding large codebases

Impact

Completing Weenix gave me a comprehensive understanding of operating system internals that has proven invaluable in my career. The project's emphasis on building robust, efficient systems code has influenced my approach to software engineering, particularly in performance-critical applications and when working close to the hardware.
